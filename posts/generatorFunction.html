# Generator function

# 什么是Generator函数?

Generator function是ES6推出的一个异步编程解决方案, 并不是一个传统意义上的function, 其内部封装了若干个状态, 每次执行next方法可以将这些状态一一取出.

# 定义Generator函数

function* gen() {
let a = (yield 1)
// yield表达式需用括号包裹, 否则容易混淆
let b = a + (yield 2)
return a + b
}

# Generator函数的执行

let iterator = gen()
iterator.next() // { value: 1, done: false }
iterator.next() // { value: 2, done: false }
iterator.next() // { value: NaN, done: true }
iterator.next() // { value: undefined, done: true }

是不是跟想象中有点不一样, 为什么第三步会返回NaN,我们修改下代码, 看看a, b到底是什么

function* gen() {
  let a = (yield 1)
  console.log('a', a)

  // yield表达式需用括号包裹, 否则容易混淆
  let b = a + (yield 2)
  console.log('b', b)
  return a + b
}

let iterator = gen()
iterator.next() // { value: 1, done: false }

iterator.next() // a undefined
                // { value: 2, done: false }

iterator.next() // b NaN
                // { value: NaN, done: true }

iterator.next() // { value: undefined, done: true }

为什么 a 是undefined呢, 难道yield表达式不会返回表达式后面的值吗?

是的, yield表达式默认不返回任何值, 或者说总返回undefined, 但是next()函数提供了一个参数, 表示将该参数当做上一次yield表达式的返回值

let iterator = gen()
iterator.next() // { value: 1, done: false }

iterator.next(1) // a 1
                // { value: 2, done: false }

iterator.next(2) // b 3
                // { value: 4, done: true }

iterator.next() // { value: undefined, done: true }
