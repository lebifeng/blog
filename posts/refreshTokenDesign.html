# refreshToken设计

熟悉前后端分离开发的小伙伴一定对JWT(Json Web Token)不陌生,这是目前流行的认证解决方案,相较于以前存储在服务端的Session, JWT最大的特点就是保持服务端无状态, 即服务端不存储会话信息, 这种特性给分布式服务带来了天然优势. 但是仅使用JWT也会带来一些麻烦: 用户正常使用系统, access_token突然过期了,此时后端返回401未授权,前端一般直接跳转到登录页面了(用户:我日...),用户体验极差. 解决办法较多,一般参考OAuth 2.0协议的refreshToken 设计,许多设计虽然能解决上述问题,却并不能很好的处理并发请求和免刷页面, 此处给出一个基于Axios的并发免刷新解决方案

# 基于Axios的并发免刷新refreshToken解决方案

基本思路:

  1. 后端认证成功后,生成两个jwt, 一个为access_token, 过期时间较短,api 请求时携带此token. 另一个为refresh_token, 过期时间较长, 有效期内携带此refreshToken调用相应api 可获取新的access_token
  2. 每次请求前判断access_token是否过期,如果过期则阻塞当前以及随后发起的api请求,再使用refreshToken换取 新的access_token,替换旧的access_token
  3. 恢复阻塞的请求

Q&A:

  1. 如何阻塞当前请求?
利用axios.interceptors.request.use(function(config | Promise) { return config
  }) 拦截每次请求, 且支持返回promise

  1. token过期后如何解决并发请求?

    节流函数

直接上代码:

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Title</title>
</head>
<body>
<button type="button" onclick="request()">请求</button>
</body>
<script type="text/javascript"
        src="../node_modules/axios/dist/axios.js"></script>
<script type="text/javascript">
  const TOKEN_KEY = "TOKEN_KEY"

  function getItem(key) {
    return localStorage.getItem(key)
  }

  function setItem(key, value) {
    return localStorage.setItem(key, value)
  }

  // 初始化一个token, 过期时间2s
  setItem(TOKEN_KEY, JSON.stringify({
    token: 'newToken',
    startTime: Date.now(),
    expires: 2 * 1000
  }))

  let gettingRefreshToken = false
  let instance = axios.create()
  let currentRequest = null

  instance.interceptors.request.use(function (config) {
    let {token, startTime, expires} = JSON.parse(getItem(TOKEN_KEY))
    config.headers.token = token

    // token 已过期
    if (startTime + expires < Date.now()) {
      // refreshToken请求不要拦截, 此处可合理判断
      if (config.url === 'http://localhost:3000/users/refreshToken') {
        return config
      }
      // 节流函数
      if (gettingRefreshToken === false) {
        gettingRefreshToken = true
        // 换取新的access_token
        currentRequest = getRefreshToken().then(res => {
          setItem(TOKEN_KEY,
              JSON.stringify({
                token: res.data,
                startTime: Date.now(),
                expires: 5 * 1000
              }))
          gettingRefreshToken = false
          return config
        })
        console.log('refreshToken 请求已发起')
        return currentRequest
      } else {
        console.log('别催了, refreshToken 请求已经发起了, 拿到新token马上就恢复')
        return currentRequest.then(() => {
          return config
        })
      }
    } else {
      console.log('这是正常请求')
      return config
    }
  })

  let num = 0

  function request() {
    num++
    return instance.get('http://localhost:3000/users/user?id=' + num)
    .then(res => {
      console.log(`这是第${res.data}个请求`)
    })
    .catch(function (thrown) {
      console.error(thrown)
    })
  }

  // refreshToken请求, 为了简单演示这里就不传access_token了
  function getRefreshToken() {
    return instance.get('http://localhost:3000/users/refreshToken')
  }
</script>
</html>
