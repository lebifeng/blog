# HTTP缓存

# 强缓存 和 协商缓存

强缓存: 缓存有效期内直接命中浏览器缓存, 无需亲请求服务器

协商缓存: 命中缓存但已失效或者服务器明确要求验证缓存有效性, 此时必须发起请求向服务器验证缓存有效性

# 缓存有关的几个Header

expires: http 1.0 用来设置资源的有效时间, 是一个GMT时间格式字符串, 表示资源在此之前可用, 由于时间是服务器设置的, 客户端验证缓存有效性时, 如果时间与服务器有偏差, 会导致缓存有效期不准确, 甚至缓存一直处于过期状态

cache-control: 为了解决expires的问题, http 1.1 添加该字段, 用来表示资源缓存策略和有效时间, 常用参数为: cache-control: public|private max-age=3000s no-cache|no-store must-revalidate

  • public: 表示客户端和代理端都能缓存

  • private: 表示仅客户端能缓存

  • max-age: 资源有效时间, 单位秒

  • no-cache: 始终要向服务器验证缓存有效性

  • no-store: 不允许使用缓存

  • must-revalidate: 资源过期才向服务器验证缓存有效性

last-modified: 资源最后修改时间, 服务器用来判别资源是否修改,用在响应中

Etag: 服务端为资源生成的唯一标注, 可用来判别资源是否修改,用在响应中

if-no-match: 浏览器使用协商缓存时, 将 Etag 字段值放在此字段发送到服务端

if-modified-since: 浏览器使用协商缓存时, 将 last-modified 字段值放在此字段发送到服务端
