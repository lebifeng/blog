# VUE自定义组件之inheritAttrs

# 使用自定义组件时, 大家有没有用过inheritAttrs属性? 知道这个属性的用处吗?来跟我一探究竟吧

# 正常声明一个VUE组件

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Title</title>
  <script type="text/javascript" src="../node_modules/vue/dist/vue.js"></script>
</head>
<body>
<div id="root">
  <index style="font-style: italic" class="class2" :real-prop="1999" :data-nums="888"
         data-age="24"></index>
</div>
</body>
<script type="text/javascript">
  Vue.component('index', {
    // inheritAttrs: false,
    data() {
      return {
        name: 'index'
      }
    },
    props: ['real-prop'],
    template:
          `
      <div class="class1" style="color: red" data-nums="999">
        <span>this is span</span>
      </div>
    `
  })
  let vm = new Vue(
      {
        el: '#root'
      }
  )
</script>
</html>

我们定义的组件自带class和style属性, 并且有一个初始data-nums属性, 在使用index组件时, 我们给组件添加了real-prop, 并且重写了class和style属性值, 也重写了自定义属性data-nums的属性值, 那么页面输出结果是什么呢

可以看到, real-prop没有出现在div属性中, data-nums属性也被重写, 我们添加的data-age属性也出现了, 这符合预期, 但是class和style标签的属性值却被合并了. 其实, 这是vue为class属性和style属性约定的特殊规则: 它们会合并而不是覆盖

再来看加上inheritAttrs: false, 会发生什么

发现除了class和style标签, 我们声明的data-nums属性和data-age属性并没有添加到组件元素上, 在联想vue实例的$attrs属性, 不难猜测: 组件实例化时, 会将那些不属于prop和class以及style的属性统统放到$attrs对象 里,然后判断inheritAttrs属性是否为false, 不为false的话, 会将这些attrs属性绑定到组件的根元素上,否则,不做任何绑定.

为了验证以上猜测,我们打印一下$attrs的值, 以及手动绑定$attrs对象到根元素上,看看情况如何

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Title</title>
  <script type="text/javascript" src="../node_modules/vue/dist/vue.js"></script>
</head>
<body>
<div id="root">
  <index style="font-style: italic" class="class2" :real-prop="1999" :data-nums="888"
         data-age="24"></index>
</div>
</body>
<script type="text/javascript">
  Vue.component('index', {
    inheritAttrs: false,
    data() {
      return {
        name: 'index'
      }
    },
    mounted() {
      console.log(this.$attrs)
    },
    props: ['real-prop'],
    template:
          `
      <div v-bind="$attrs" class="class1" style="color: red" data-nums="999">
        <span>this is span</span>
      </div>
    `
  })
  let vm = new Vue(
      {
        el: '#root'
      }
  )
</script>
</html>

输出$attrs

手动绑定$attrs后的页面

可以看到, data-nums属性 和data-age 属性回来了,但是data-nums属性值仍然是999,并没有被使用组件时声明的属性所覆盖 这是怎么回事? 随即意识到, 可能是$attrs和原始data-nums属性声明的顺序问题导致的,将v-bind="$attrs"移动到最后面, 在看结果, 问题应该迎刃而解了吧.

template:
          `
      <div class="class1" style="color: red" data-nums="999"  v-bind="$attrs">
        <span>this is span</span>
      </div>
    `

what fuck! 还是999

好吧, 暂时得出一个结论, 使用inheritAttrs:false时, 手动绑定$attrs并不能覆盖组件原有的非特殊(prop, class, style)属性 , 不知道为什么会这样, 哪天有空研究下源码.

其实inheritAttrs一般是配合$attrs , 将属性赋给组件内需要的元素,例如:

Vue.component('base-input', {
  inheritAttrs: false,
  props: ['label', 'value'],
  template: `
    <label>
      {{ label }}
      <input
        v-bind="$attrs"
        v-bind:value="value"
        v-on:input="$emit('input', $event.target.value)"
      >
    </label>
  `
})
