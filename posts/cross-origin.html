# 常见跨域解决方案

# 什么是跨域? 跨域是服务器拦截了响应还是浏览器拦截了?

了解跨域之前, 先了解下同源策略: http(s): //www.abc.com:8081, 一个网址, 一般由三个部分组成, 协议、域名、端口. 这三者完全相同的两个网址称之为同源, 否则就是跨域. 注意, 一般http的默认端口是80, https则是443, 默认均不展示. 当浏览器发起一个请求时, 会判断该请求是否为跨域请求, 如果是跨域请求, 则不会将服务器返回的响应交给用户处理(服务器会正常返回响应), 因此, 跨域请求的响应是被浏览器拦截了, 这是出于安全考虑.

# 如何解决跨域?

一般有三种方法: CORS, JSONP, 反向代理

# CORS

cors 是 cross-origin resource share的简称, 意为跨域资源共享, 简单理解就是服务器设置相应cors响应头来告诉 浏览器你不要管我的跨域请求 使用方法参考 跨域解决方案

# 反向代理

既然浏览器认为请求跨域了,我们可以使用一个反向代理服务器作为服务器的代理, 只要把该反向代理服务器的网址设置和客户端同源, 浏览器就不会认为请求跨域了.

客户端(http://abc.com:8081) <--- 跨域 --- 服务端(http://xyz.com:8088)

客户端(http://abc.com:8081) <---正常请求--- 反向代理服务器(http://abc.com:8081) <---代理--- 服务端(http://xyz.com :8088)

# JSONP

简单来说就是利用js脚本等资源的下载不受同源策略影响, 我们发起一个jsonp请求时, 实际上就是手动生成一个<script>标签, 给src赋上我们想要请求的api, 服务端将资源作为js内容返回给我们.

那么我们如何获取服务端返回的内容呢?使用回调函数

例如:客户端和服务端事先约定好一个回调函数

function callback(data) {
    console.log('data is:', data)
}

那么服务端 返回的js内容就是 callback('你好啊')

js资源一下载完就会执行, 此时<script>标签内的内容为:

<script>
    callback('你好啊')
</script>

当然就是执行我们事先定义好的callback函数,数据自然就拿到了
