# 递归优化

上节实现fibonacci函数用到了递归法,并且提到使用递归所带来的的缺点, 那么如果实际工作中必须要用到递归呢? 我们可以使用尾递归来优化

# 尾调用函数

尾调用指一个函数的最后一步是调用一个函数,比如:

// 尾调用
function f(x){
  return g(x);
}

// 以下都不是尾调用
// 情况一
function f(x){
  let y = g(x);
  return y;
}

// 情况二
function f(x){
  return g(x) + 1; // 除了调用函数之外还参与了表达式计算
}

尾调用由于是函数的最后一步操作，所以不需要保留外层函数的调用记录，因为调用位置、内部变量等信息 都不会再用到了，只要直接用内层函数的调用记录，取代外层函数的调用记录就可以了。因此,尾调用能优化函数 执行性能

# 尾递归优化

递归函数最后一步调用自身称为尾递归,递归非常耗费内存，因为需要同时保存成千上百个调用记录， 很容易发生"栈溢出"错误（stack overflow）。但对于尾递归来说，由于只存在一个调用记录， 所以永远不会发生"栈溢出"错误。

尾递归优化改写fibonacci函数


// 原始fibonacci函数
function fibonacci(n) {
  if (n === 0) {
    return 0
  }
  if (n === 1 || n === 2) {
    return 1
  }

  return fibonacci(n - 1) + fibonacci(n - 2)
}

// 使用尾递归思想的改进fibonacci函数
function improvedFibonacci(n, pre = 0, next = 1) {
  if (n === 0) {
    return 0
  }
  if (n === 1) {
    return next
  }
  return improvedFibonacci(n - 1, next, pre + next)
}

注意

尾调用优化只在严格模式下起作用, 非严格模式下 由于有 arguments 和 callee等参数的存在,无法优化调用栈,并且实际 测试中, chrome似乎没有进行尾递归优化
